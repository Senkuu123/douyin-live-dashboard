# 抖音直播实时数据看板

一款运行在Windows上的抖音直播监控软件

输入正在直播的房间号后，软件会持续采集直播间互动数据，并在一个页面内展示在线人数、进场、弹幕、点赞、礼物、用户事件、观众问题和运营提示

数据保存在你自己的MySQL数据库中，停止采集后可以导出本场CSV文件

> 当前版本：`v0.1.0`
>
> 适用于公开直播间的实时观察、直播复盘和本地数据分析

## 软件能做什么

- 查看当前在线人数和最近15分钟的在线趋势
- 统计进场人数、弹幕数、点赞数、礼物数和独立用户数
- 查看各项互动的每分钟速率
- 实时滚动展示弹幕和用户互动事件
- 记录礼物名称、数量、送礼用户和钻石价值
- 标记礼物高峰、弹幕问题激增和在线下降
- 识别优惠券、价格、链接、库存、物流、售后等常见观众问题
- 根据直播数据生成行动提示
- 查看发言榜、送礼榜和粉丝团榜
- 按最低用户等级筛选全局互动数据
- 调整直播态势各项评分权重
- 导出本次采集会话的CSV数据
- 软件重新启动后恢复最近一次有数据的会话

## 运行前准备

使用软件前需要准备：

- Windows 10或Windows 11，x64系统
- MySQL 8.0数据库
- 能正常打开目标抖音直播间的网络环境

软件是便携版，不需要安装。采集组件已经包含在EXE中，不需要单独下载

## 下载软件

在GitHub Releases中下载：

```text
DouyinLiveDashboard-0.1.0-x64.exe
```

建议同时核对发行页面提供的SHA-256校验值

当前版本没有商业代码签名，Windows可能显示SmartScreen提示。确认文件来自本仓库发行页面后，可以选择仍要运行

## 首次配置数据库

软件会把直播数据写入MySQL项目库`douyin_live_dashboard`

在当前Windows用户目录创建`.my.cnf`文件，例如：

```text
C:\Users\你的用户名\.my.cnf
```

文件内容：

```ini
[client]
host=127.0.0.1
port=3306
user=你的数据库用户
password=你的数据库密码
```

如果使用远程MySQL，把`host`和`port`改成对应地址

数据库账号需要具备以下权限：

- 连接MySQL
- 创建和使用`douyin_live_dashboard`
- 创建项目数据表
- 读取和写入项目数据

数据库密码只保存在本机`.my.cnf`中，不需要填写到软件界面

## 开始采集

1. 双击运行`DouyinLiveDashboard-0.1.0-x64.exe`
2. 在顶部房间号输入框填写正在直播的抖音房间号
3. 也可以先复制完整直播间链接，再点击输入框右侧的粘贴图标
4. 点击开始
5. 状态变为采集中后，等待数据进入看板

直播间地址示例：

```text
https://live.douyin.com/123456789
```

对应房间号：

```text
123456789
```

点击停止会结束本次采集会话。再次开始会创建一个新会话，不会接着写入上一场会话

## 界面功能

### 顶部状态栏

顶部状态栏显示：

- 房间号
- 直播间标题
- 采集中或待命
- 已连接时长
- 重连次数
- 最后更新时间
- 开始或停止
- 用户等级筛选
- CSV导出

### 直播态势

直播态势根据以下数据计算实时评分：

- 当前在线人数
- 进场速率
- 弹幕速率
- 点赞速率
- 礼物速率
- 新增独立用户速率
- 高价值礼物数
- 高等级用户进场数
- 负面词控制情况

点击设置可以调整各项权重，所有权重合计必须为100

三个详情入口分别用于查看：

- 高价值礼物及送出用户
- 高等级用户进场详情
- 负面词和对应弹幕内容

### 核心指标

看板顶部显示六项指标：

| 指标     | 显示内容                         |
| -------- | -------------------------------- |
| 当前在线 | 当前直播间在线人数和近5分钟变化  |
| 进场人数 | 本场进场总数和每分钟进场速率     |
| 弹幕数   | 本场弹幕总数和每分钟弹幕速率     |
| 点赞数   | 本场点赞总数和每分钟点赞速率     |
| 礼物数   | 本场礼物总数和每分钟礼物速率     |
| 独立用户 | 本场不重复用户数和每分钟新增速率 |

每张指标卡下方的小图显示对应数据最近一段时间的变化

### 在线人数趋势

趋势图每分钟更新一次，显示最近15分钟的在线人数

图中可能出现以下标签：

| 标签         | 触发条件                                        |
| ------------ | ----------------------------------------------- |
| 礼物高峰     | 本分钟不少于5件礼物，并达到前3分钟均值的2倍     |
| 弹幕问题激增 | 本分钟不少于3条问题弹幕，并达到前3分钟均值的2倍 |
| 在线下降     | 较上一分钟减少至少20人，并且降幅达到5%          |

同一类标签至少间隔2分钟。标签生成后会固定在对应时间点，直到随15分钟窗口移出

把鼠标停在事件竖线上，可以查看该分钟在线人数和触发标签的数据

### 行动提示

行动提示会根据高等级用户进场、高价值礼物、负面词和在线下降生成卡片

点击设置可以选择：

- 是否聚合同一分钟内的同类事件
- 是否显示建议动作

关闭聚合后，每个触发事件会单独显示一张提示卡片

### 实时弹幕

实时弹幕显示用户等级、昵称、弹幕原文和时间

底部开关可以控制：

- 自动滚动
- 高亮高等级用户

用户等级每10级使用一种颜色，等级越高越醒目

### 高价值事件流

事件流展示礼物、进场、点赞、关注、分享和粉丝团事件

顶部可以切换全部、礼物、进场、点赞和关注

打开只看高价值后，只保留高价值礼物、高等级用户进场、大额点赞和关注等事件

### 问题队列

问题队列以弹幕原文作为标题，自动识别以下常见问题：

- 优惠活动
- 价格咨询
- 商品链接
- 库存和尺码
- 物流和发货
- 售后和退款
- 商品质量
- 使用方法
- 规格参数
- 赠品权益
- 直播体验
- 其他问题

相同弹幕会合并显示出现次数和最近时间

### 榜单

榜单支持三种排行：

- 发言榜：按本场弹幕数量排序
- 送礼榜：按本场礼物钻石价值排序
- 粉丝团榜：显示粉丝团等级和本场采集到的互动次数

## 用户等级筛选

点击顶部筛选按钮，可以选择最低用户等级

应用筛选后，以下模块只统计该等级及以上用户产生的数据：

- 进场、弹幕、点赞、礼物和独立用户
- 互动速率
- 直播态势评分
- 实时弹幕
- 高价值事件流
- 问题队列
- 行动提示
- 榜单

当前在线人数和在线趋势属于整个直播间的数据，不按用户等级拆分

## 导出数据

点击顶部导出按钮，可以把本次采集会话保存为CSV文件

导出内容包括：

- 事件类型
- 采集时间
- 匿名用户哈希
- 用户昵称
- 弹幕或事件内容
- 用户等级
- 礼物名称、数量和钻石价值
- 点赞数量
- 在线人数
- 采集方式和采集版本

CSV中不会包含原始用户ID

## 礼物采集不到怎么办

部分直播间的匿名连接不会返回礼物消息。这种情况可以给采集组件提供本机抖音登录态

仓库提供了[Cookie转Base64URL工具](scripts/cookie_to_base64url.py)，用于把浏览器Cookie转换为软件需要的Base64URL格式。脚本只使用Python标准库，输入内容不会显示在终端，也不会写入文件

在仓库根目录运行：

```powershell
python scripts/cookie_to_base64url.py
```

按提示粘贴Cookie并回车，将输出结果填入下方`DOUYIN_COOKIE_B64`。Base64URL只是编码，不会加密Cookie，转换结果仍应按敏感凭据保管

在EXE同目录创建`.env.local`：

```dotenv
DOUYIN_COOKIE_B64=<Base64URL编码后的Cookie>
```

配置完成后完全退出软件，再重新启动并开始一场新采集

使用登录态时需要了解：

- Base64URL只是编码，不是加密
- `.env.local`只应保存在本机
- 不要把Cookie上传到GitHub、Issue、截图或日志
- Cookie失效后需要重新配置
- 礼物消息是否返回仍可能受到直播间和平台风控影响

## 常见问题

### 点击开始后没有数据

检查以下项目：

1. 房间号是否正确
2. 直播间是否正在直播
3. 当前网络能否正常打开该直播间
4. MySQL是否正在运行
5. `.my.cnf`地址、端口、用户名和密码是否正确
6. 数据库账号是否有项目库和项目表的创建权限

### 直播间标题一直等待获取

标题需要等直播间返回有效房间信息。确认直播间正在直播，并观察连接状态和最后更新时间

### 礼物数一直为0

先确认直播间实际发生了送礼事件。如果其他互动正常但礼物始终为0，再按礼物采集不到怎么办配置登录态

### 在线人数长时间不变化

在线人数由直播间返回的实时房间数据决定，趋势图只在完整分钟结束后更新一次

### 软件停止后数据还在吗

数据已经写入MySQL。重新打开软件会恢复最近一个有数据的历史会话

### Windows阻止软件运行

当前版本没有商业代码签名，Windows可能显示SmartScreen提示。确认文件来自本仓库发行页面，并核对SHA-256校验值

## 数据和隐私

- 数据保存在用户配置的MySQL数据库中
- 原始用户ID不会明文入库，只保存SHA-256哈希
- Cookie、数据库密码和token不会写入Git仓库
- CSV导出不包含原始用户ID
- 软件不会自动评论、点赞、关注或发送直播互动

## 当前限制

- 同一时间只能采集一个直播间
- 网络波动、直播结束、登录失效和平台限流可能造成数据缺口
- 问题队列使用固定关键词分类，可能出现漏判或分类不准确
- 采集依赖抖音网页版协议，平台调整后可能需要升级软件
- 不保证所有直播间在匿名模式下都能收到礼物消息
- 当前版本只提供Windows x64便携版

## 使用范围

软件只用于公开直播间的数据观察和本地分析，不绕过验证码、登录限制或访问控制，不提供批量账号、自动评论和自动互动功能

使用者需要自行遵守当地法律、平台规则和数据处理要求
